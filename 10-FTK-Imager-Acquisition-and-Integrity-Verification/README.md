# Digital Forensics: FTK Imager Acquisition and Integrity Verification

This project demonstrates the technical process of installing AccessData FTK Imager on a Kali Linux system using Wine and performing a forensically sound logical drive acquisition.

## Project Objective
The goal was to establish a functional forensic imaging environment in Linux and execute a bit-for-bit acquisition of a logical drive while maintaining evidence integrity through cryptographic verification.

## Tools and Environment
* **Operating System:** Kali Linux (Rolling)
* **Software:** AccessData FTK Imager v4.7.3.81
* **Compatibility Layer:** Wine (Wine Is Not an Emulator)
* **Acquisition Format:** Raw (dd)

## Implementation Details

### Environment Configuration
The forensic tool was executed within a dedicated Wine environment. Technical hurdles such as RPC service communication errors and system-wide file access through the `Z:` drive mapping were resolved to ensure full functionality.

### Acquisition Process
1. **Source Selection:** A logical drive was chosen as the evidence source.
2. **Imaging:** A bit-for-bit copy was created using the Raw (dd) format to ensure maximum compatibility for further analysis.
3. **Storage:** Evidence was directed to a separate partition to prevent data contamination on the source media.

### Integrity and Verification
The primary focus of the lab was to ensure that the digital evidence remained unaltered. FTK Imager calculated MD5 and SHA1 hashes during acquisition and compared them against the final image file.

**Verification Results:**
* **MD5:** ae0edc55245538a82f9c0c982020efeb (Verified Match)
* **SHA1:** 66fbf665d267c587f69062f4c6da901580de5fb4 (Verified Match)

The verified match confirms that the acquisition is a perfect replica of the original data.

* Deployment of Windows-based forensic tools on Linux systems.

* Resource management and troubleshooting in a forensic workstation environment.
